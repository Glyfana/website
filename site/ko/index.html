기능

설치를 요청하기 전에, 패키지된 앱이 실제로 무엇을 하는지 분명하게 보여줍니다.

Glyfana는 단순한 설치 파일이 아닙니다. 이 영역은 패키지된 빌드에서 제공되는 편집 흐름, 이미지 처리, 복구, 업데이트, 검증 동작을 사용자에게 설명합니다.

편집 흐름

실제 소스 모드를 갖춘 Typora 스타일 마크다운 편집.

기본 화면은 렌더링 결과에 가깝게 유지되지만, 정밀한 편집이 필요할 때는 언제든 원본 Markdown으로 내려갈 수 있습니다.

  • Milkdown과 ProseMirror 기반의 WYSIWYG 작성 화면
  • CodeMirror 6 소스 모드로 일반 Markdown 즉시 전환
  • Shiki 기반 코드 블록 하이라이팅으로 기술 문서 가독성 유지

이미지 워크플로우

붙여넣은 이미지를 잃어버리지 않고 노트 옆에 함께 보관합니다.

로컬 이미지는 앱 내부 저장소에 숨기지 않고 문서 옆에 저장해서, 파일 자체를 이동해도 휴대성이 유지됩니다.

  • 이미지를 바로 드래그 앤 드롭하거나 붙여넣기 가능
  • 저장된 파일은 같은 위치의 <note>.assets/ 폴더에 이미지 저장
  • 상대 경로와 mdlocal:// 렌더링으로 개발/패키지 빌드 모두에서 미리보기 유지

자동 저장

저장된 문서와 제목 없는 초안 모두 별도의 복구 경로를 가집니다.

앱은 문서가 실제 파일에 연결되어 있는지에 따라 다른 idle autosave 규칙을 적용합니다.

  • 저장된 문서는 편집기가 쉬는 동안 자동으로 디스크에 반영
  • 제목 없는 문서는 앱 userData 폴더에 draft 형태로 보존
  • 충돌 복구와 draft 복원으로 세션 사이 데이터 손실을 완화

파일 안전성

저장되지 않은 변경과 외부 수정 충돌을 우선적으로 다룹니다.

Glyfana는 디스크 복사본이 항상 우선이라고 가정하지 않습니다. 로컬 작업이 덮이지 않도록 선택지를 먼저 보여줍니다.

  • 저장되지 않은 변경이 있으면 창 닫기나 다른 파일 열기 전에 경고
  • 문서를 열어 둔 동안 외부 파일 변경 감지
  • 디스크에서 다시 읽기, 계속 편집, 다른 이름으로 저장, 의도적 덮어쓰기 지원

업데이트

설치 이후에도 GitHub 기반 업데이트 흐름이 이어집니다.

패키지 앱은 이 사이트가 참고하는 동일한 릴리스 채널을 추적할 수 있어서, 사용자가 새 빌드를 수동으로 찾지 않아도 됩니다.

  • 패키지된 빌드는 GitHub Releases 기반 자동 업데이트 가능
  • 앱 시작 시와 이후 1시간마다 백그라운드 확인
  • 도움말 메뉴와 재시작 프롬프트로 수동/인앱 업데이트 흐름 제공

검증

설치 전에 신뢰 신호를 화면 위에 남겨 둡니다.

사용자는 무엇이 바뀌었는지, 어떤 자산이 올라왔는지, 설치 전에 어떻게 검증할지 한 페이지에서 확인할 수 있습니다.

  • 메인 CTA는 현재 안정 Setup.exe 빌드를 직접 가리킴
  • 릴리스 노트, 자산 목록, SHA256 값을 한 화면에서 제공
  • GitHub API가 불가능할 때도 고정 manifest 데이터 유지

설치

추가 패키징 인프라 없이 Windows 설치 파일을 배포합니다.

GitHub Releases는 그대로 산출물 저장소로 사용하고, 이 페이지는 다운로드, 설치, 검증 안내를 한 곳에서 정리한 진입점을 제공합니다.

파일 해시 검증

설치 전에 로컬 SHA256 값을 이 페이지에 공개된 체크섬과 비교합니다.

설치 마법사 실행

FAQ

설치 전에 나올 질문을 먼저 답합니다.

망설임을 줄이는 가장 빠른 방법은 Glyfana가 파일, 복구, 업데이트, 검증을 어떻게 다루는지 평이한 말로 설명하는 것입니다.

노트에 이미지를 붙여넣으면 어떻게 되나요?

Glyfana는 로컬 이미지를 바로 붙여넣거나 드롭할 수 있습니다. 저장된 노트의 경우 이미지 파일은 같은 위치의 <note>.assets/ 폴더에 저장되고 상대 경로로 참조되어 문서 휴대성이 유지됩니다.

저장하지 않은 채 앱을 닫으면 어떻게 되나요?

로컬 변경이 남아 있으면 창을 닫거나 다른 파일로 전환하기 전에 경고가 표시됩니다. 저장된 문서는 idle autosave를 쓰고, 제목 없는 문서는 앱 데이터 폴더에 draft 형태로 보존되어 다음 세션에서 복구할 수 있습니다.

같은 파일이 Glyfana 밖에서 바뀌면 어떻게 되나요?

Glyfana는 외부 파일 변경을 감지하고 조용히 덮어쓰지 않습니다. 사용자는 디스크에서 다시 읽기, 계속 편집, 다른 이름으로 저장, 의도적으로 덮어쓰기 중 하나를 선택할 수 있습니다.

설치 이후 업데이트는 어떻게 들어오나요?

패키지 빌드는 앱 시작 시와 이후 1시간마다 GitHub Releases를 확인할 수 있습니다. 도움말 메뉴에서 수동 확인도 가능하고, 새 빌드가 준비되면 인앱 프롬프트에서 재시작 흐름까지 이어집니다.

설치 전에 파일 검증은 어떻게 하나요?

Setup.exe를 내려받은 뒤, 검증 섹션의 PowerShell 명령으로 SHA256 값을 계산해서 이 페이지에 표시된 값과 비교하면 됩니다.

검증

빌드를 쉽게 신뢰할 수 있게 만듭니다.

체크섬, PowerShell 검증 명령, 공개된 모든 자산 목록을 설치 전에 바로 확인할 수 있게 제공합니다.

무결성

PowerShell로 설치 파일 해시를 계산해 공개된 SHA256 값과 비교하세요.

SHA256 대기 중

PowerShell Get-FileHash ".\installer.exe" -Algorithm SHA256
