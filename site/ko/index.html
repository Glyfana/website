최신 변경점

다운로드 페이지 안에서 최신 릴리스 핵심 내용을 바로 확인할 수 있습니다.

GitHub에 릴리스 노트가 게시되면 이 영역에서 중요한 줄을 먼저 보여주고, 전체 노트는 한 번에 열 수 있게 연결합니다.

기능

설치 버튼을 누르기 전에, 패키징된 앱이 실제로 무엇을 하는지 먼저 설명합니다.

Glyfana는 단순한 설치 파일이 아닙니다. 이 섹션에서는 에디터 흐름, 이미지 처리, 복구, 업데이트, 검증 방식까지 패키지 앱이 제공하는 기능을 구체적으로 정리합니다.

편집 흐름

Typora 스타일 WYSIWYG 편집과 실제 소스 모드를 함께 제공합니다.

기본 화면은 렌더링된 문서에 가깝게 유지하면서도, 정밀한 수정이 필요할 때는 원문 Markdown으로 바로 내려갈 수 있습니다.

  • Milkdown과 ProseMirror 기반의 WYSIWYG Markdown 편집
  • CodeMirror 6 소스 모드 전환으로 원문 직접 수정 가능
  • Shiki 기반 코드 블록 하이라이트로 기술 문서 가독성 유지

이미지 워크플로

이미지를 드롭하거나 붙여넣고, 노트 옆에 자산을 정리합니다.

로컬 이미지를 앱 내부에 숨기지 않고 문서 옆에 두기 때문에, 파일을 옮겨도 자산 구조를 추적하기 쉽습니다.

  • 드래그 앤 드롭이나 붙여넣기로 이미지를 바로 삽입
  • 저장된 파일은 같은 위치의 <note>.assets/ 폴더에 자산 저장
  • 상대 경로와 mdlocal:// 렌더링으로 개발 환경과 패키지 앱 모두에서 미리보기 유지

자동 저장

저장된 문서와 새 초안 각각에 맞는 복구 경로가 있습니다.

문서가 실제 파일에 연결돼 있는지 여부에 따라 idle autosave 동작이 달라져, 작업 상태에 맞는 복구가 이뤄집니다.

  • 저장 경로가 있는 문서는 유휴 시 자동으로 디스크에 저장
  • 새 문서는 앱 userData에 초안 형태로 자동 저장
  • 앱 종료나 크래시 이후에도 초안 복구와 복원 흐름 제공

파일 안전성

저장하지 않은 변경과 외부 수정 충돌을 별도 상태로 다룹니다.

디스크 파일이 항상 우선이라고 가정하지 않고, 로컬 작업이 덮어써지기 전에 선택지를 먼저 보여줍니다.

  • 저장 안 된 변경이 있으면 창 닫기나 다른 파일 열기 전에 경고
  • 문서가 열려 있는 동안 외부 파일 변경을 감시
  • 다시 불러오기, 계속 편집, 다른 이름으로 저장, 덮어쓰기 중에서 선택 가능

업데이트

설치 후에도 GitHub 기반 업데이트 흐름이 이어집니다.

이 웹사이트가 보는 릴리스 채널을 앱도 계속 추적할 수 있으므로, 사용자는 새 빌드를 수동으로 찾지 않아도 됩니다.

  • 패키징된 빌드는 GitHub Releases 기반 자동 업데이트 가능
  • 앱 시작 시와 이후 매시간 백그라운드 업데이트 확인
  • Help 메뉴 확인과 재시작 프롬프트로 수동/인앱 업데이트 모두 지원

검증

설치 전에 신뢰 근거를 같은 페이지에 남깁니다.

사용자는 설치를 실행하기 전에 어떤 변경이 있었는지, 어떤 자산이 올라왔는지, 해시는 무엇인지 한 번에 확인할 수 있습니다.

  • 메인 버튼은 현재 안정 Setup.exe 설치 파일을 가리킴
  • 릴리스 노트, 자산 목록, SHA256을 같은 페이지에서 확인 가능
  • 라이브 GitHub API가 실패해도 pinned manifest로 기본 정보 유지

설치

추가 배포 인프라 없이도 Windows 설치 파일을 배포할 수 있습니다.

GitHub Releases는 그대로 아티팩트 저장소로 두고, 이 페이지는 다운로드, 설치, 검증 안내를 한 번에 제공하는 더 나은 진입점 역할을 합니다.

01

현재 설치 파일 다운로드

기본 버튼으로 현재 안정 릴리스에 연결된 .exe 설치 파일을 바로 받을 수 있습니다.

02

파일 해시 검증

설치 전에 로컬 SHA256 값과 공개된 체크섬을 비교해 파일 무결성을 확인합니다.

03

설치 후 변경 사항 확인

릴리스 노트와 전체 자산 목록이 연결되어 있어 배포 내용 전체를 빠르게 검토할 수 있습니다.

검증

설치 전에 신뢰 근거를 먼저 보여줍니다.

체크섬, PowerShell 검증 명령, 공개된 릴리스 자산을 설치 시작 전에 한 자리에서 확인할 수 있게 합니다.

무결성

PowerShell로 설치 파일 해시를 계산해 공개된 SHA256 값과 비교하세요.

SHA256 대기 중

PowerShell Get-FileHash ".\installer.exe" -Algorithm SHA256
